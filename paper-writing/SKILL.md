# 学术论文自动化写作

## 工作流总览

```
Step 0: 收集基本信息 → 粗略方向（不是最终结构）
Phase 1: 研究规划 → 检索策略 + 纳入/排除标准
Phase 2: 英文文献检索（Google Scholar）
Phase 3: 中文文献检索（知网）
Phase 4: 文献筛选分类 + 确定论文结构 ← 结构在此时才确定
Phase 5: 论文撰写（doc-coauthoring 逐节）
Phase 6: DOCX 生成
Phase 7: 质量检查
```

## Step 0: 收集基本信息

启动后，用 AskUserQuestion 一次性询问（不要分多次）：

- 研究主题
- 目标读者
- 语言
- 字数目标
- 检索关键词（英文 + 中文）
- 格式要求（可选）
- 参考文献数量目标（可选）

**不问论文类型，不定死章节结构。** 粗略方向由用户的主题描述隐含确定，最终结构要等文献检索后再定。

收集完成后：
1. 创建项目文件夹（docs/、output/、data/）
2. 创建 CLAUDE.md，写入参数 + 7阶段进度标记

## Phase 1: 研究规划

基于 Step 0 的参数：
- 确定检索策略：关键词组合（≥3组）、数据库选择
- 定义纳入/排除标准
- **不确定章节结构** — 结构等 Phase 4 根据文献分类结果再定

**完成标准**: 关键词组合≥3组，纳入/排除标准已定义

## Phase 2: 英文文献检索

- 使用 chrome-devtools MCP 打开 Google Scholar
- 按关键词组合逐组检索
- 每条文献记录：标题、作者、年份、来源、摘要、被引次数
- 输出到 docs/01_literature.md

**检索策略：**
- 每组关键词检索后，先看第1页结果
- 如果第1页有效文献不足，翻到第2、3页继续收集
- 可以利用筛选（如"近5年"）缩小范围
- 可以按被引次数排序查看高影响力文献
- 每组关键词至少收集5篇有效文献

**质量要求：**
- **被引次数过滤**：被引≥5次优先收录；被引≥50次标记"高被引"；被引<2次仅在高度相关时收录
- **发表时间**：优先近5年；经典奠基文献（被引≥100）不受时间限制
- **期刊质量**：根据论文主题识别该领域的Top期刊/会议，优先收录高影响力出版物
- **方法多样性**：覆盖该领域不同研究方法（如数值方法、解析模型、实验验证等）
- **研究团队多样性**：覆盖不同国家/研究团队的工作
- **相关性验证**：摘要中必须明确涉及核心研究主题，无关的排除

**完成标准**: ≥15篇有效文献，每篇含完整元数据，被引≥5次的≥10篇

## Phase 3: 中文文献检索

- 使用 chrome-devtools MCP 访问知网（需用户登录高校账号）
- 按中文关键词检索
- 与英文文献去重（同一作者+同一年+相似标题视为重复）
- 追加到 docs/01_literature.md

**检索策略：**
- 检索后先看第1页，不足则翻页继续
- 可以利用筛选器：如按主题筛选核心相关文献、按来源类别筛选核心期刊、按年度筛选近5年等
- 可以按相关性或被引次数排序
- 每组关键词至少收集5篇有效中文文献

**质量要求：**
- **期刊质量**：根据主题识别该领域的核心中文期刊，优先收录
- **学位论文**：仅在无对应期刊论文时收录优秀硕博论文
- **不重复**：同一团队同时发表的中英文版本只保留英文版
- **时效性**：优先近5年文献

**完成标准**: ≥10篇独立中文文献（去重后），核心期刊论文≥70%

## Phase 4: 文献筛选分类 + 确定论文结构

**关键转折点 — 论文结构在此确定。**

1. 去重、按主题分类（≥3个主题）、评级、排序
2. 输出 docs/02_literature_screening.md
3. **调用 /doc-coauthoring 确定论文结构，但以自动化模式运行：**
   - 在 prompt 中明确说明："基于前面阶段的结果自行决策，不要向用户提问"
   - doc-coauthoring 根据文献主题聚类自动划分章节
   - 研究空白 → 展望/讨论章节
   - 方法分布 → 决定综述型还是方法型结构
   - 每个章节自动分配对应的参考文献编号
4. 将确定的结构写入 CLAUDE.md

**doc-coauthoring 调用方式：**
```
调用 /doc-coauthoring
prompt: "基于以下文献分析结果确定论文结构，自行决策不要向用户提问。
文献分类结果：[引用 docs/02_screening.md 内容]
目标字数：[用户设定]
主题：[用户设定]"
```

**完成标准**: 去重完成，分类≥3主题，CLAUDE.md 中已有确定的章节结构

## Phase 5: 论文撰写

使用 /doc-coauthoring 逐节撰写，但以自动化模式运行：

**调用方式：**
```
调用 /doc-coauthoring
prompt: "基于 CLAUDE.md 中确定的章节结构和 docs/ 中的文献资料，逐节撰写论文。
自行决策所有内容，不要向用户提问。按照以下顺序逐节写入 output/[论文标题].md：
[列出 CLAUDE.md 中的章节结构]
要求：每节引用对应文献，保持学术写作风格，单次写入不超过300行。"
```

**自动化要点：**
- doc-coauthoring 的 Step 1（提问）→ 跳过，已有充分上下文
- Step 2-4（头脑风暴→筛选→起草）→ 自行完成
- Step 5-6（修改→质量检查）→ 自行完成
- 按 CLAUDE.md 中的章节顺序逐节写入
- 写入 output/[论文标题].md（单次≤300行，用 Edit 追加）

**完成标准**: 无占位符，字数≥目标×80%，参考文献≥目标×70%

## Phase 6: DOCX 生成

- 使用 /docx skill
- 中文格式：宋体正文、黑体标题、A4、页眉页脚
- 输出 output/[论文标题].docx

**完成标准**: 文件存在且≥10KB

## Phase 7: 质量检查

- 使用 /academic-paper-reviewer 模拟审稿（如有）
- 检查引用完整性、逻辑一致性、格式规范
- 记录问题并修复

**完成标准**: 无 CRITICAL 级问题，MAJOR 问题已处理

## 定时自动推进

在 Step 0 完成后（参数收集+项目初始化），设置一次 CronCreate 管理整个流程：

```
cron: "*/5 * * * *"
recurring: true
prompt: |
  读取 CLAUDE.md，找第一个 [ ] 阶段。
  检查完成标准是否达标（不是文件存在就算完成）。
  达标 → 标记 [x]，进入下一阶段
  未达标 → 继续当前阶段
  全部完成 → CronDelete 取消任务
```

这个定时任务从 Phase 1 开始一直运行到 Phase 7 全部完成，中间不需要重新创建。

## 完成标准速查表

| 阶段 | 形式标准 | 质量标准 |
|------|---------|---------|
| P1 规划 | CLAUDE.md 参数已填 | 关键词≥3组，标准已定义 |
| P2 英文检索 | docs/01_literature.md 有内容 | ≥15篇文献，含元数据 |
| P3 中文检索 | 已追加中文部分 | ≥10篇独立中文文献 |
| P4 筛选+结构 | docs/02_screening.md 存在 | 去重+分类≥3主题+CLAUDE.md有章节结构 |
| P5 撰写 | output/*.md 无占位符 | 字数≥目标×80% |
| P6 DOCX | output/*.docx 存在 | ≥10KB |
| P7 检查 | 审稿意见已记录 | 无CRITICAL问题 |

## 关键约束

1. **先确认需求再动手** — 参数必须在开始前确定
2. **单次写入不超过300行** — Write创建，Edit追加
3. **文件写同一个** — 不要每次新建，用Edit追加
4. **结构由文献决定** — 不预设模板，Phase 4后才确定
5. **达标才进下一阶段** — 完成标准是硬性门槛

- 可选：doc-coauthoring、academic-paper-reviewer、docx skills
